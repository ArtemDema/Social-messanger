## <a id="мета-розробки-цього-додатку">Мета розробки цього додатку:</a>
Даний проєкт створений для того, щоб користувачі могли без обмежень обмінюватися інформацією — зображеннями, текстами та іншими даними.

## <a id="технології-проекту">Технології проекта:</a>
### Python
#### Головна мова програмування у проекті – Відповідає повністю за бекенд
1. Django - фреймворк для бекенд-розробки. У проєкті відповідає за рендеринг сторінок, створення API-ендпоінтів та взаємодію з базою даних.
2. Pillow (PIL) - бібліотека для обробки медіафайлів користувачів.
3. channels - модуль для реалізації асинхронних вебсокет-з’єднань на стороні бекенду.
4. daphne - сервер, що забезпечує асинхронну роботу Django.
### SQL
#### Мова для роботи з реляційними базами даних, інтегрована з Django ORM.
### HTML
#### Використовується для створення структури вебсторінок. Контент частково формується через Django-шаблони та JavaScript.
### CSS
#### Відповідає за стилізацію вебсторінок.
### JS
#### Забезпечує інтерактивність на стороні клієнта (відкриття форм, робота вебсокетів, пінги тощо).
### Git
#### Система контролю версій, що використовувалась для командної розробки.
### Figma
#### Інструмент для створення дизайну інтерфейсів та планування структури проєкту.

## <a id="інструкції-по-запуску-проекту">Інструкції по запуску проекту:</a>

### Як запустити проект ЛОКАЛЬНО

1. >Переконайтесь, що встановлено Python 3.10+ та pip
2. >Завантажте проєкт, натиснувши кнопку "<> Code" → "Download ZIP"
3. >Розпакуйте архів у зручну папку
4. >Відкрийте термінал у директорії проєкту та перейдіть у неї через cd 
5. >У папці socialmessanger виконайте:
```bash
    pip install -f requirements.txt
    # Це встановить всі залежності у проекта
```
6. >Перейдіть у директорію, де знаходиться manage.py (за допомогою команди "cd")
7. >Створіть та застосуйте міграції:
```bash
    python manage.py makemigrations
    # Це створить міграції для бази даних.
    
    python manage.py migrate
    # Це проведе міграції бази даних - створить всі моделі проекту та зробе його базу даних працюючою
```   
8. >Запустість проект:
```bash
    python manage.py runserver
    # Це запустить проект локально
```

## <a id="додатки">Додатки:</a>
    admin.py - Відповідає за реєстрацію та налаштування моделей у адмін-панелі

    apps.py - Містить конфігурацію Django-додатку

    models.py - Описує структуру бази даних (таблиці та зв’язки)

    urls.py - Визначає маршрути та зв’язує їх з обробниками

    forms.py - Відповідає за створення та валідацію форм

    consumers.py - Реалізує WebSocket-класи для постійного зв’язку з клієнтом

    routing.py - Описує маршрути для WebSocket-з’єднань

    views.py - Містить логіку обробки HTTP-запитів і формування відповідей

    templates - Папка з HTML-шаблонами сторінок

        *.html - Шаблони вебсторінок


project - Папка з основною конфігурацією проєкту

    asgi.py - Налаштування асинхронної роботи (WebSocket, ASGI)

    settings.py - Основні налаштування Django проєкту

    urls.py - Глобальні маршрути сайту та медіафайлів

    wsgi.py - Синхронний інтерфейс для запуску Django


static - Папка зі статичними файлами (JS/CSS/зображення)

    *_app - Статичні файли конкретного додатку

        js - JavaScript-скрипти

            script.js - Основний скрипт

        css - стилі

            styles.css - Основні стилі

        images - статичні зображення сайту

        fonts - шрифти

            *.ttf - файли шрифтів


media - Папка для збереження файлів, завантажених користувачами. Файли мають унікальні імена для уникнення конфліктів.

manage.py - Основний файл для керування Django-проєктом

README.md - Поточний файл з описом проєкту

requirements.txt - Список залежностей для встановлення

.gitignore - Перелік файлів і папок, які ігноруються Git

### Home_app 
- Відповідає за головну сторінку
- Тут користувач може переглядати стрічку постів, свої пости, а також бачити запити в друзі та повідомлення
- У розділі власних постів відображаються тільки власні публікації для зручного керування

### Post_app 
- Відповідає за сторінку «Мої публікації»
- Дозволяє переглядати та керувати власними постами

### Friends_app
- Реалізує систему друзів: запити, список друзів та рекомендації
- Дозволяє фільтрувати користувачів за категоріями: друзі, запити, рекомендації

### User_app
- Відповідає за реєстрацію, авторизацію та профіль користувача
- Після реєстрації надсилається підтвердження на email, що обмежує один акаунт на одну пошту

### Chat_app
- Реалізує чат-систему в реальному часі
- Дозволяє обмінюватися текстом та зображеннями
- Підтримує групові чати з керуванням адміністратора
- Адмін може змінювати налаштування групи, додавати/видаляти учасників та видаляти групу
- Уся взаємодія відбувається через WebSocket без перезавантаження сторінки

### Settings_app
- Строрінка для налаштувань профілю
- На цій сторінці можно дотати інформацію про: День народження, ім'я, призвище, змінити пошту та змінити свій юз
- Підтримує групові чати з керуванням адміністратора

## Детальні особливості додатків

### Зображення

Файли зображень зберігаються в папці media та прив’язуються до моделей бази даних. Вони повністю керуються користувачами: можуть створюватися, змінюватися та видалятися.

### Система друзів

Дружба формується після відправки та прийняття запиту. У разі відхилення запиту користувачі залишаються не пов’язаними.

Навіть після завершення дружби користувачі можуть спілкуватися, якщо чат не був видалений.

### Чати

Чат-система реалізована повністю на WebSocket, що дозволяє працювати без перезавантаження сторінки.

Користувачі можуть надсилати повідомлення та зображення, а також створювати групові чати.

На домашній сторінці відображаються останні повідомлення в чатах.

Поки користувач онлайн, він відмічається як «у мережі».

## <a id="висновок-роботи">Висновок роботи:</a>

### Висновок

Під час виконання проєкту було розроблено прототип соціальної мережі з підтримкою функцій реального часу. Застосування Django у поєднанні з Django Channels та сервером Daphne дозволило реалізувати двосторонню взаємодію між користувачами, що є ключовим елементом сучасних соціальних платформ. Завдяки цьому вдалося впровадити живі чати, миттєві сповіщення та динамічне оновлення даних без необхідності перезавантаження сторінки, що зробило користувацький досвід більш плавним і комфортним.

Проєкт створювався командою з трьох розробників. Злагоджена робота команди та ефективна комунікація дозволили успішно подолати технічні труднощі й забезпечити стабільність продукту. У процесі розробки використовувалися системи контролю версій та інструменти організації задач, що сприяло впорядкованому робочому процесу та швидкому вирішенню проблем.
